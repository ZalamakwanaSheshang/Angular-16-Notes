Template-Driven Forms

Template-driven forms are the simpler way to build forms in Angular. The form logic (like validation rules and control definitions) is primarily handled in the template using Angular directives. They are suitable for simple forms (e.g., login forms) and rely heavily on two-way data binding with the [(ngModel)] syntax.
You must import the FormsModule in your Angular module (e.g., app.module.ts) to use template-driven forms.

Key Concepts:

  • ngModel Directive: Creates an instance of FormControl internally and binds it to a form element. It's essential for two-way data binding.
  • name Attribute: Required on all form controls using ngModel to register the control with the parent NgForm and provide the key for the form value object.
  • NgForm Directive: Automatically attached to the <form> tag by the FormsModule. It manages all form controls within the form, tracks the overall form state (e.g., valid, invalid, dirty), and provides the form value.
