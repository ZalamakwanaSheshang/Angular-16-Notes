Data Binding

Data binding in Angular is a powerful feature that allows you to synchronize data between the component and the view. This ensures that any changes in the component's state are automatically reflected in the view and vice versa. Angular provides several types of data binding, each serving different purposes and use cases.

Types of Data Binding

One-Way Binding

One-way binding allows data to flow in a single direction, either from the component to the view or from the view to the component.

From Component to View

This type of binding is achieved using Interpolation and Property Binding.
Interpolation:
It allows you to embed expressions within string literals in the view. Angular evaluates these expressions and updates the view accordingly.
For example: <h1> { {title} } </h1>

Property Binding:
It binds a property of an HTML element to a property in the component. This is done using square brackets [].
For example: <img [src]="imageUrl">
This is an example of Property Binding

From View to Component

This type of binding is achieved using Event Binding.
Event Binding:
It binds an event in the view to a method in the component. This is done using parentheses ().
For example: <button (click)="onSave()">Save</button>
This is an example of Event Binding.

Two-Way Binding

Two-way binding allows data to flow in both directions, from the component to the view and from the view to the component. This is particularly useful in forms where user input needs to be reflected in the component and vice versa.

Two-Way Binding:
It is achieved using the [(ngModel)] directive.
For example: <input [(ngModel)]="name">
